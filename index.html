    SideKick

    A voice-first portal to your agent.

    Toolbar

    • Speak — toggles TTS playback of agent replies. Tap to mute.
    • Lock (pocket lock) — covers the screen while streaming stays live; swipe the slider left-to-right to unlock. Handy for bike / pocket use.
    • Info — this panel.
    • Settings — model, STT/TTS engine, voice, etc.

    Composer

    • New chat — clears transcript and starts a fresh session.
    • Camera / Attach — attach image(s) up to 5 MB. Disabled if the active model doesn't support images.
    • Voice input — tap the mic to record; tap chevron for options. Toggle Push to talk for press-and-hold, Call mode for live streaming, and Auto-send to ship the transcript without a review step.
    • Send — green when anything is ready to send (typed text, memo, transcript, or attachment).

    Tips

    Commit word (Settings): saying the configured word — default "over" — auto-sends the transcript while streaming. Blank = disabled.

    Install as a PWA

    Installing gives SideKick its own home-screen icon, runs fullscreen without browser chrome, and on iOS enables the Pocket Lock background-audio path.

    • iPhone / iPad (Safari): tap the Share button → Add to Home Screen. Other iOS browsers can't install PWAs — Apple only permits this via Safari.
    • Android (Chrome / Edge / Firefox): three-dot menu → Install app, or Add to Home Screen.
    • Desktop Chrome / Edge: install icon in the address bar, or three-dot menu → Install SideKick.
    • Desktop Safari: File → Add to Dock (macOS Sonoma 14+).

    Caveats

    • Phone lock (power button): iOS suspends mic capture while truly locked. Pocket Lock is the workaround for bike/pocket use.
    • iOS premium voices (Evan, Alison…): downloadable via Settings → Accessibility → Spoken Content, but they're reserved for native apps / VoiceOver — Web Speech on iOS only exposes the built-in voice bank (Samantha, Daniel, etc.).
    • Desktop Chrome + local TTS: Chromium has a bug where SpeechSynthesis fires silently (no audio, no error) when other audio streams are active. We've tried several workarounds without success — use Server TTS or Safari on this combination.
